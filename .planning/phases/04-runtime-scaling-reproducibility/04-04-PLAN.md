<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PyPI resolver -> release job | External package resolution determines code executed in the scheduled release environment. |
| workflow report -> step summary | Generated runtime metrics are rendered into human-readable GitHub summary output. |
| CI matrix -> support signal | Compatibility audit results could be mistaken for a declared Python support policy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering | scheduled dependency install | mitigate | Use committed pip constraints in release install and cache invalidation per D-15/RUN-04. |
| T-04-04-02 | Elevation of Privilege | GitHub Actions jobs | mitigate | Preserve Phase 03 least-privilege job permissions; audit job uses read-only permissions. |
| T-04-04-03 | Repudiation | Python compatibility audit | mitigate | Name and test a separate audit job while keeping `requires-python = ">=3.14"` unchanged per D-17. |
| T-04-04-04 | Denial of Service | runtime metrics in workflow | mitigate | Append compact metrics only; do not add runtime warning thresholds or hard gates per D-12/D-13. |
| T-04-SC | Tampering | pip constraints and installs | mitigate | No new packages are introduced; 04-RESEARCH package audit reports no `[SUS]` or `[SLOP]` packages, and workflow installs only through pip-native constraints. |
</threat_model>
